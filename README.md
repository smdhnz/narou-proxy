# Narou Proxy

「小説家になろう」の作品を、より快適なUIで閲覧するためのプロキシアプリケーションです。
Next.js (App Router) と Bun を使用して構築されており、自宅サーバーなどのローカル環境での運用を想定しています。

## 特徴

- **高速な閲覧**: 余計な広告やスクリプトを排除し、本文のみを抽出して表示します。
- **レスポンシブデザイン**: PC、タブレット、スマートフォンのすべてのデバイスで最適化された表示を提供します。
- **読書設定**:
  - **フォント切り替え**: ゴシック体 / 明朝体の選択。
  - **背景・文字色**: ダーク / ライト / セピア の3テーマを用意。
  - **自動非表示ヘッダー**: タップで表示・非表示をトグルでき、5秒後に自動で隠れる没入型UI。
- **読書履歴**: 閲覧した小説の履歴をブラウザに保存し、続きから読みやすくします。

## 技術スタック

- **Framework**: Next.js 16 (App Router / Standalone mode)
- **Runtime**: Bun
- **Styling**: Tailwind CSS / shadcn/ui
- **Parsing**: Cheerio

## セットアップ (Docker)

自宅サーバーなどで Docker を使用して起動する場合：

1. リポジトリをクローンまたは配置します。
2. 以下のコマンドを実行してビルドと起動を行います。

```bash
docker compose up -d --build
```

3. ブラウザで `http://<サーバーのIP>:3000` にアクセスします。

### 外部公開 (SSH Tunnel)

`docker-compose.yml` には、`tcpexposer.com` を利用して外部からアクセス可能にするための SSH トンネル設定が含まれています。

- **必要条件**: ホストの `~/.ssh/id_ed25519` に適切な SSH 秘密鍵が必要です。
- **仕組み**: `narou-proxy-tunnel` サービスが自動的に起動し、指定されたサブドメインでアプリを公開します。
- **注意**: 公開する場合は、[注意事項] を必ず確認し、自己責任で行ってください。

### Docker イメージの最適化

本プロジェクトは Next.js の `standalone` 出力を活用したマルチステージビルドを採用しています。
これにより、イメージサイズを **約300MB程度**（従来比 約70%以上削減）まで軽量化し、リソースの節約とデプロイの高速化を実現しています。

## ローカル開発

```bash
# 依存関係のインストール
bun install

# 開発サーバーの起動
bun run dev
```

## 注意事項（重要）

- **利用規約について**: このアプリケーションは「小説家になろう」のサイトを直接スクレイピングして情報を取得しています。公式API（なろうデベロッパー）の範囲外の動作を含むため、**不特定多数への公開は避け、個人利用の範囲に留めてください。**
- **IPブロックのリスク**: 短時間に大量のリクエストを送信すると、運営側からアクセス制限（IPブロック）を受ける可能性があります。
- **著作権**: 小説の著作権は各作者に帰属します。コンテンツの二次配布などは行わないでください。
